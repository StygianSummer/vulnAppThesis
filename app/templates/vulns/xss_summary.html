Summary: Cross-site Scripting (XSS)

Cross-site scripting (XSS) is a web vulnerability that allows attackers to inject malicious scripts into websites. When viewed by users, the aforementioned malicious scripts can be used to steal private data, impersonate users, and modify the content of the page without the user’s knowledge.

There are 3 main types of XSS attacks, namely:

  • Reflected XSS: Here, an attacker injects malicious code onto a website through an input field. This input is reflected immediately in the response, and can be triggered when a user clicks on a crafted URL.
  • Stored XSS: The malicious input in this type of XSS attack is stored on the server (such as in a database) and executed when any user opens the page affected by the script. This is the most dangerous type of XSS, as it can harm all users who visit the affected page.
  • DOM-based XSS: The attack happens entirely on the client-side when JavaScript manipulates unsafe user input.

Demo: What went wrong?

  • The user input was directly inserted into the HTML using an f-string.
  • No escaping or data validation was done on the input. This, in turn, allowed scripts to be executed.

Fixing the problem:

  • User input should not be trusted. It is important to always sanitize and encode the input before rendering it on a page.
  • In the fix part of the demo, html.escape() was used to neutralize special characters in user input

There are many ways to avoid different XSS scripting attacks. For a more in depth understanding of this vulnerability, please consult the optional reading below.
