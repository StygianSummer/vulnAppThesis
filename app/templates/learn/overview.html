Vulnerabilities

What is a Vulnerability?

In cybersecurity, a vulnerability is a flaw or a weakness in a system that can be taken advantage of by a malicious entity (like a hacker!) to compromise its security. These vulnerabilities when taken advantage of (or exploited) lead to unfavorable outcomes and cause significant loss to organisations and people. These unfavorable outcomes include:

  • stealing personal or sensitive data
  • gaining unauthorised access
  • damaging or disrupting operations

Despite best efforts, virtually all software and hardware contain bugs or mistakes. A vulnerability is a bug that creates a security risk. Not all vulnerabilities are equally dangerous. The risk level of a vulnerability depends on the nature of the vulnerability and well as the value of the surrounding system.

Some vulnerabilities can only be used for denial of service (DoS) attacks to crash a website, more dangerous ones allow the attacker to run their own harmful code (called malware), without the user being aware of it.

One Important category of vulnerabilities is a Web vulnerability.

Web Vulnerabilities

Web Applications are programs that run on websites, such as online stores, social media platforms, or login systems. They are inherently less secure than other applications. Additionally, they often interact with users and databases and are a common target for attackers.

Web Vulnerabilities are system flaws and weaknesses in these web-based applications. These vulnerabilities, when exploited, lead to data breaches, and other security incidents such as malicious entities gaining unauthorized access, manipulating data, or disrupting services.

OWASP Top Ten is a well-known awareness document for developers that highlights the most critical security risks to web applications. This demo focuses on three of them, namely:

  • SQL Injection
  • Cross-Site Scripting (XSS)
  • Cryptographic Failures
