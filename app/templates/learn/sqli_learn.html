SQL Injection

What is SQL?

Most websites and applications that store information use SQL (Structured Query Language) to work with databases.

For example, SQL is used to get data: 

  • SELECT * FROM Customers;
    gets all the records from the "Customers" table.

There are SQL statements that can do all sorts of things.

  • SELECT gets data from a database
  • INSERT INTO adds new data
  • UPDATE changes existing data
  • DELETE removes data
  • CREATE TABLE creates a new table
  • DROP TABLE deletes a table

What is SQL Injection?

SQL Injection is a security vulnerability. When a website asks for user input like a username, email, or search term it often uses that input to build an SQL statement. When a user searches for something in an application, an SQL statement runs an SQL statement to get some data. .

For example, a website might use this SQL to find a user:

  • SELECT * FROM Users WHERE Username = 'user_input';

If the user types Krishna, it becomes:

  • SELECT * FROM Users WHERE Username = 'Krishna';

But, if someone types something malicious like:

  • ' OR 1=1 --

The SQL becomes:

  • SELECT * FROM Users WHERE Username = '' OR 1=1 --';

The problem:

  • OR 1=1 is always true, so it returns all users instead of just one.

  • -- tells SQL to ignore the rest of the query (it’s a comment).

This process is called SQL Injection—the attacker “injects” their own SQL code into the query.

Why It's Dangerous

With SQL injection, attackers can do things including but not limited to:

  • Log in without a password
  • Access all user data
  • Delete or change information
  • Destroy the whole database
